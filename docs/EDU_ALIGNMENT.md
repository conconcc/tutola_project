# EDU_ALIGNMENT.md — TUTOLA as an Education Service Solution

> 이 문서는 TUTOLA가 **차세대 교육 서비스 솔루션** 콘테스트의 요구 조건을 어떻게 충족하는지를 명시한다.
> 심사자와 팀원 모두 이 문서를 통해 제품의 교육 서비스적 가치를 이해할 수 있어야 한다.

---

## 1. 핵심 재정의: TUTOLA는 실습형 교육 모듈 플랫폼이다

TUTOLA는 단순한 생활 가이드 앱이 아니다.

> **TUTOLA = Adaptive Skill Training Platform**
> 생활·취미 도메인(커피 브루잉, 세탁, 요리)을 소재로 삼아,
> **단계 기반 실습 → AI 코치 피드백 → 계획 재구성**의 흐름을 통해
> 기능적 스킬을 체계적으로 학습할 수 있는 교육형 플랫폼이다.

### 왜 이 소재인가?

| 이유 | 설명 |
|------|------|
| **낮은 진입 장벽** | 누구나 경험하는 영역 → 학습 동기 확보 용이 |
| **실습 중심** | 지식 전달이 아닌 "직접 해보는" 구조 → 교육적 효과 높음 |
| **확장성** | 동일 엔진으로 바리스타 자격, 세탁사 훈련, 조리사 실습 과정으로 확장 가능 |
| **측정 가능** | 각 step의 완료/오류/소요 시간이 데이터로 기록 → 학습 분석 가능 |

## 2. 교육 현장 혁신 (Classroom Innovation)

### 문제
기존 실습 교육은 강사 1명이 다수 학습자를 동시에 지도하기 어렵고,
학습자의 현재 숙련도나 보유 자원 조건이 모두 다른데 획일적인 절차를 따른다.

### TUTOLA의 해결

| 기능 | 교육적 효과 |
|------|------------|
| **조건 기반 계획 생성** | 학습자마다 다른 보유 장비/재료에 맞는 맞춤 실습 계획 제공 |
| **Step Drill-Down** | 특정 단계에서 막힌 학습자가 스스로 하위 단계를 탐색 |
| **3D 시각화 (커피)** | 실물이 없어도 시점 전환으로 공간적 이해 확보 |
| **Warning 시스템** | 실수하기 전에 선제적 경고 → 안전하고 효율적인 실습 |
| **AI 코치 채팅** | "이 단계 다시 설명해줘" → 강사 없이도 즉각 피드백 |

## 3. 운영 행정 혁신 (Operational Innovation)

### 문제
교육 기관은 수강생 진도 관리, 출석, 평가, 보고서 작성에 많은 행정 비용을 쏟는다.

### TUTOLA의 해결

```
ProgressEvent 자동 기록
  → 각 step 완료 시 timestamp, stepId, eventType 자동 저장
  → 강사는 대시보드에서 전체 수강생 진도를 한눈에 파악

AssessmentRecord 자동 생성
  → 실습 완료 시 점수화 가능한 기록 자동 생성
  → 수동 평가지 작성 불필요

Cohort 관리
  → 기관 단위로 코호트 생성 및 학습자 목록 관리
  → 일정 기반 과정 운영 지원
```

**예상 행정 비용 절감:**
- 진도 확인 시간 → 자동화
- 평가 기록 작성 → 자동화
- 맞춤 자료 준비 → AI 계획 생성으로 대체

## 4. 수강생 경험 혁신 (Learner Experience Innovation)

### 기존 교육 경험의 한계

| 문제 | 현상 |
|------|------|
| 일방향 전달 | 강사가 설명, 학습자는 수동적으로 듣고 따라함 |
| 획일적 속도 | 빠른 학습자와 느린 학습자 모두 동일한 속도 강요 |
| 맥락 없는 반복 | "왜 이렇게 하는지" 설명 없이 절차만 반복 |
| 피드백 지연 | 실수해도 강사가 발견하기 전까지 수정 불가 |

### TUTOLA가 제공하는 새로운 경험

```
학습자가 자신의 조건을 입력 (재료, 도구, 수량)
    ↓
AI 코치가 현재 조건에 맞는 계획을 재구성
    ↓
단계별로 진행하며 막히면 drill-down으로 세분화
    ↓
자연어로 "이 단계 더 자세히" → AI가 즉각 응답
    ↓
완료 후 다음에 더 잘할 수 있는 조언 제공
```

**핵심 경험 원칙:**
- 정답형 튜토리얼이 아닌 **현재 상황 맞춤형 안내**
- 코치형 말투: 비난 없이, 행동 중심으로
- 자기 주도 학습 가능: 강사 없이도 완주 가능한 구조

## 5. 교육 비즈니스 모델 (B2B Education Business Model)

### Tier 1 — 개인 사용자 (B2C)
- 무료 체험: 시나리오 3개 기본 제공
- 향후 프리미엄: 고급 시나리오, 실황 강사 연결, 전문가 피드백

### Tier 2 — 교육 기관 라이선스 (B2B)

| 대상 기관 | 활용 방식 |
|----------|----------|
| 바리스타 훈련 학원 | 커피 시나리오를 기관 커리큘럼에 삽입 |
| 직업훈련원 | 조리사 실습 모듈로 활용 |
| 기업 복지 교육 | 사내 웰니스 프로그램(요리, 세탁)으로 도입 |
| 온라인 교육 플랫폼 | 화이트 라벨 SDK로 기존 LMS에 임베드 |

### Tier 3 — API / 화이트레이블 (B2B2C)
- 시나리오 엔진 API 라이선스
- 타 플랫폼에 TUTOLA 엔진 삽입

## 6. 콘테스트 심사 기준 대응 요약

| 심사 기준 | TUTOLA의 대응 |
|----------|--------------|
| **교육 현장 혁신** | 조건 맞춤 계획, Drill-Down, 3D 시각화, Warning 선제 안내 |
| **운영 행정 혁신** | ProgressEvent 자동 기록, AssessmentRecord, Cohort 관리 |
| **수강생 경험 혁신** | AI 코치 채팅, 자연어 명령, 자기주도 학습 구조 |
| **비즈니스 모델** | B2C 무료체험 + B2B 기관 라이선스 + B2B2C API |

## 7. 확장 로드맵 (Post-MVP)

```
MVP
└── 커피 / 빨래 / 요리 3개 시나리오
└── 공통 시나리오 엔진
└── rule-based AI 코치

↓ Phase 2
└── Course / Module 구조 추가
└── AssessmentRecord 자동 생성
└── 기관 대시보드 (강사 뷰)

↓ Phase 3
└── LLM 기반 AI 코치 고도화
└── 음성 인식 연동
└── 카메라 비전 (실습 동작 인식)
└── 외부 LMS 임베드 API

↓ Phase 4
└── 새 도메인 시나리오 추가 (자동차 정비, 반려동물 케어 등)
└── 강사 마켓플레이스
└── 수강생 포트폴리오 시스템
```
